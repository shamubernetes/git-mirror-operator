# Security Policy

## Reporting a Vulnerability

Do not open public issues for vulnerabilities.

Use GitHub private vulnerability reporting for this repository if it is enabled. If it is not enabled, contact the repository maintainers privately through the owning organization before sharing exploit details publicly.

Include:

- Affected version or commit.
- A clear description of the impact.
- Reproduction steps or proof of concept when safe to share privately.
- Any known mitigations.

## Supported Versions

This project is pre-1.0. Security fixes are expected to land on `main` and in the next tagged release.
